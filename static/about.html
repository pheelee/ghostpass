About GhostPass

GhostPass is a minimal, privacy-focused service for sharing sensitive information securely. It uses client-side encryption to ensure that your secrets remain private and are automatically destroyed after viewing.

Security Features

  • Client-side encryption - Your secret is encrypted in your browser before being sent to the server
  • AES-GCM encryption - Industry-standard encryption with a unique 256-bit key per secret
  • Key in URL fragment - The encryption key is stored in the URL fragment (#) and never sent to the server
  • One-time retrieval - Secrets are deleted immediately after being viewed
  • Auto-expiry - Secrets automatically expire after the chosen time period
  • IP restrictions - Optionally restrict access to specific IP ranges
  • Rate limiting - Protection against brute force and abuse
  • No logs - Minimal data retention with no logging of secret content

How the Encryption Works

  1. When you create a secret, your browser generates a random 256-bit encryption key
  2. The secret is encrypted using AES-GCM with a unique initialization vector
  3. Only the encrypted data (ciphertext and IV) is sent to the server
  4. The encryption key is encoded and added to the URL as a fragment (after #)
  5. URL fragments are never sent to the server by browsers, keeping the key private
  6. When the recipient opens the link, their browser extracts the key from the fragment
  7. The encrypted data is fetched from the server and decrypted locally
  8. After viewing, the secret is permanently deleted from the server

Data Stored on Server

The server only stores:

  • A random unique ID for the secret
  • The encrypted ciphertext (unreadable without the key)
  • The initialization vector (IV) for AES-GCM
  • Expiry timestamp
  • View count and maximum views allowed
  • Allowed CIDRs (if IP restriction is enabled)

The server never stores or sees:

  • The encryption key
  • The plaintext secret content
  • Any identifying information about the creator

Open Source

GhostPass is built with transparency in mind. The entire codebase is designed to be auditable and self-hostable. You can run your own instance to maintain complete control over your secrets.

Technical Stack

  • Backend: Go with SQLite database
  • Frontend: Vanilla HTML/JS (no external dependencies)
  • Encryption: Web Crypto API (AES-GCM)
  • Transport: HTTPS with security headers

Important Security Notes

  • Always share the secret link over a secure channel
  • Anyone with the full link (including the #key) can decrypt the secret
  • The secret is only as secure as the channel you use to share the link
  • Consider using end-to-end encrypted messaging for sharing links
  • Once viewed, the secret is permanently destroyed
